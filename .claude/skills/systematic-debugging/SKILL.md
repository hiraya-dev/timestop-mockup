# Systematic Debugging

1. Reproduce the failure and capture the exact error/output.
2. Read the failing test/log/stack before touching code.
3. Form a hypothesis, verify it with targeted inspection or instrumentation.
4. Fix the root cause, not the symptom; then re-run the failing check plus the tier-appropriate verification.
