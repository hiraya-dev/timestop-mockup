# Brainstorming

Before writing or changing an app spec:

1. Restate the request and reference material in product terms.
2. Decide canvas sizing mode, panels, media/upload flow, control sections, export/copy behavior, renderer technique, and timeline/layers need.
3. Resolve ambiguity with explicit recorded decisions; only ask the user for product-critical ambiguity.
4. Output a concise spec covering behavior, controls, state mapping, and acceptance-relevant entities.
