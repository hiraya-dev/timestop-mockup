# Writing Plans

From an approved spec, produce a step-by-step plan that:

1. Lists exact files to edit (schema, renderer, routes, acceptance, performance, worklog).
2. Orders work so each step is verifiable (typecheck, unit, browser).
3. Names the verification tier and the commands to run.
4. Avoids open questions — every decision is fixed before code is edited.
